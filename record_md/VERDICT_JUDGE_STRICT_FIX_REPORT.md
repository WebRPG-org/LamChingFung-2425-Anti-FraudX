# 🎮 勝負判定系統修復報告 (嚴格版)

## 📋 問題分析

**症狀**: 無論說什麼，騙徒都會贏

**根本原因**: `verdict_judge.py` 的判定邏輯太寬鬆
- ❌ 只要提到關鍵字就算贏
- ❌ 沒有檢查是否真的提供了敏感信息或採取了行動
- ❌ 沒有區分假設和真實行動

---

## ✅ 修復內容

### 1. 騙徒勝利判定 (嚴格化)

**需要同時滿足**:
1. ✅ 包含關鍵字 (密碼、帳號、驗證碼、轉賬、身份證)
2. ✅ 包含實際數據 (4位以上數字或字母)
3. ✅ 沒有負面指示詞 (如果、假設、朋友、聽說)
4. ✅ 信心度 >= 0.8

**例子**:
- ✅ "我的密碼是 123456" → 騙徒贏
- ✅ "我已經轉了 $5000" → 騙徒贏
- ❌ "如果我提供密碼會怎樣" → 遊戲繼續
- ❌ "我朋友的帳號是..." → 遊戲繼續

### 2. 專家勝利判定 (嚴格化)

**需要同時滿足**:
1. ✅ 包含關鍵字 (報警、求助、官方確認、停止對話、懷疑)
2. ✅ 包含完成指示詞 (已經、正在、現在等) - **除了懷疑**
3. ✅ 沒有負面指示詞 (如果、假設)
4. ✅ 信心度 >= 0.8

**例子**:
- ✅ "我已經報警了" → 專家贏
- ✅ "我正在打電話報警" → 專家贏
- ✅ "我不相信你" → 專家贏 (懷疑不需要完成時態)
- ✅ "我會告訴我爸爸" → 專家贏
- ❌ "如果我報警會怎樣" → 遊戲繼續
- ❌ "我想報警" → 遊戲繼續

## 🎯 新的勝利條件

### 騙徒勝利 (嚴格)

```
關鍵字 + 實際數據 + 無負面詞 + 高信心度 = 騙徒贏
```

**關鍵字**:
- 銀行密碼 (需要4位以上數字或字母)
- 銀行帳號 (需要10位以上數字)
- 驗證碼 (需要4-6位數字)
- 轉賬 (需要完成時態 + 金額)
- 身份證 (需要身份證格式)

**負面詞** (出現則不算贏):
- 如果、假設、想、可能、朋友、聽說

### 專家勝利 (嚴格)

```
關鍵字 + 完成指示詞 + 無負面詞 + 高信心度 = 專家贏
```

**關鍵字**:
- 報警 (需要完成指示詞)
- 停止對話 (需要完成指示詞)
- 官方確認 (需要完成指示詞)
- 求助 (需要完成指示詞)
- 懷疑 (不需要完成指示詞)

**完成指示詞** (除了懷疑):
- 已經、已、完成、done、正在、現在、打、撥

**負面詞** (出現則不算贏):
- 如果、假設、想、可能
