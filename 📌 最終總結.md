## 🎯 **推薦使用方式**

### ⭐ 最佳實踐：只使用 RPG Maker 功能

這是最穩定、最推薦的使用方式：

#### 1. **使用 RPG Maker 插件**

複製插件到 RPG Maker：
```
rpg_maker_plugins/AI_Bridge.js → RPG Maker/js/plugins/
rpg_maker_plugins/AntiFraudGame.js → RPG Maker/js/plugins/
```

在 RPG Maker 中：
- 工具 → 插件管理器 → 添加並啟用
- 在事件中使用插件命令

詳細教程：`RPG_MAKER_整合指南.md`

#### 2. **使用 API**

訪問 API 文檔：`http://localhost:8000/docs`

主要端點：
- `POST /chat` - AI 聊天
- `POST /api/game/start` - 開始遊戲
- `POST /api/game/message` - 遊戲對話
- `POST /api/game/end` - 結束遊戲

## ⚠️ **已知問題**

### AI-Agent 原有功能的 Bug

**問題**：循環訓練功能有錯誤
- `Runner.run() missing 1 required keyword-only argument: 'session_id'`
- `App name mismatch detected`

**影響**：
- ❌ AI-Agent 的訓練循環功能
- ❌ WebSocket 實時對話
- ✅ RPG Maker 功能不受影響

**解決方案**：
- 只使用 RPG Maker 相關功能
- 不要使用 Web UI 的訓練功能
- 詳見：`⚠️ 已知問題說明.md`
