### 2. 動態重寫原因注入機制 ✅

**文件**: `backend/utils/rewrite_context_injector.py`

**問題**:
之前重寫時只是簡單重試，模型不知道為什麼失敗，容易重複同樣的錯誤。

**解決方案**:
- 從日誌WARNING中提取失敗原因（使用正則表達式）
- 動態構建包含失敗原因的重寫prompt
- 提供針對性的修正指導

**支持的錯誤類型**:

**騙徒錯誤**:
- `expose_identity`: 暴露騙局身份
- `seek_help`: 語氣像在求助
- `warn_victim`: 勸阻受害者
- `repeat_content`: 機械式重複

**專家錯誤**:
- `request_sensitive_info`: 要求提供敏感資料
- `too_technical`: 過於技術性，沒有安撫情緒
- `no_evidence`: 缺乏具體證據
- `late_intervention`: 介入時機太遲

**使用示例**:
```python
from utils.rewrite_context_injector import build_rewrite_prompt_with_context

# 當檢測到重寫需求時
warning_message = "騙徒暴露骗局，需要重写"
enhanced_prompt = build_rewrite_prompt_with_context(
    original_prompt=original_prompt,
    warning_message=warning_message,
    agent_type="scammer",
    conversation_context=recent_turns
)
```

**優勢**:
- ✅ 明確告知模型失敗原因
- ✅ 提供針對性的修正指導
- ✅ 減少重複錯誤
- ✅ 提高重寫成功率

### 3. 對話歷史滑動視窗和摘要機制 ✅

**文件**: `backend/utils/conversation_summarizer.py`

**問題**:
- 日誌顯示 `prompt_len` 增長到 7858+，導致超時
- 長對話消耗大量資源，響應變慢

**解決方案**:
- **滑動視窗**: 只保留最近N輪（默認5輪）完整對話
- **自動摘要**: 將更早的對話壓縮成摘要
- **動態調整**: 根據prompt長度自動調整視窗大小

**摘要策略**:
```
前10輪對話：
騙徒主要策略：使用緊迫感策略，假冒權威身份，索取敏感資料
受害者主要反應：表現出恐慌，表示相信
專家主要行動：警告受害者，提供具體建議
```

**使用方法**:
```python
from utils.conversation_summarizer import compress_conversation_history

# 自動壓縮對話歷史
compressed_prompt = compress_conversation_history(
    conversation_history,
    additional_context="你是專業騙徒..."
)
```

**配置參數**:
- `window_size`: 保留最近N輪（默認5）
- `max_prompt_length`: 最大prompt長度（默認3000）
- `enable_summarization`: 是否啟用摘要（默認True）

**優勢**:
- ✅ 大幅減少prompt長度（通常減少50-70%）
- ✅ 避免超時問題
- ✅ 保留關鍵信息
- ✅ 提高響應速度

**效果對比**:
```
優化前: prompt_len=7858, 超時風險高
優化後: prompt_len=2341, 正常響應
```

### 4. 騙徒動態策略調整系統 ✅

**文件**: `backend/utils/scammer_strategy_manager.py`

**問題**:
之前騙徒的策略（authority, urgency）比較固定，無法根據受害者反應靈活調整。

**解決方案**:
實現8種詐騙策略，根據受害者信任度和類型動態切換。

**支持的策略**:

| 策略 | 適用場景 | 關鍵詞 |
|------|----------|--------|
| `AUTHORITY` | 權威施壓 | 法律、條例、必須、配合 |
| `URGENCY` | 製造緊迫感 | 立即、馬上、只剩、倒數 |
| `GREED` | 利誘貪念 | 回報、賺錢、機會、保證 |
| `FEAR` | 恐嚇威脅 | 警察、拘捕、凍結、調查 |
| `SYMPATHY` | 裝可憐 | 幫忙、困難、家人、求你 |
| `SOCIAL_PROOF` | 社會證明 | 好多人、大家、成功、限量 |
| `CONFUSION` | 混淆視聽 | 專業術語、複雜程序 |
| `FLATTERY` | 恭維奉承 | 聰明、有眼光、VIP、特選 |

**策略選擇邏輯**:
```python
# 根據受害者類型選擇策略
if victim_persona == "elderly":
    strategy = FEAR  # 老年人用恐嚇
elif victim_persona == "greedy":
    strategy = GREED  # 貪婪型用利誘
elif victim_persona == "overconfident":
    strategy = FLATTERY  # 過度自信者用恭維
```

**切換條件**:
- 信任度顯著下降（>10分）
- 同一策略使用3輪以上無效
- 信任度過低（<40）
- 專家佔上風（expert_trust > scammer_trust）

**使用方法**:
```python
from utils.scammer_strategy_manager import recommend_scammer_strategy

strategy = recommend_scammer_strategy(
    victim_trust_scammer=55,
    victim_trust_expert=70,
    victim_persona="average",
    previous_strategy="authority",
    trust_change_trend="decreasing"
)
# 返回: "urgency" 或 "fear" 等
```

**優勢**:
- ✅ 策略靈活多樣
- ✅ 根據實時情況調整
- ✅ 針對不同受害者類型
- ✅ 提高騙術成功率（用於測試專家）
