# AI 防詐騙 - 個人對話模式使用說明

## 功能概述

個人對話模式是一個全新的功能，允許用戶通過網頁聊天界面與 AI 防詐騙系統進行一對一對話。

### 兩種對話模式

1. **🛡️ 防詐助手模式**
   - 角色：退休警察黃sir
   - 功能：提供專業防詐建議、案例分析、風險評估
   - 適用場景：
     - 懷疑遇到詐騙，需要專業意見
     - 想學習防詐知識
     - 需要分析可疑情況

2. **🎭 騙徒模擬模式**
   - 角色：職業騙徒林志強
   - 功能：模擬真實詐騙對話，讓用戶體驗詐騙手法
   - 支持10種騙局類型：
     - WhatsApp 對話詐騙
     - 假短訊釣魚
     - 虛假投資應用程式
     - 假網站冒充銀行
     - 刷單騙案
     - 中獎詐騙
     - 假冒官員詐騙
     - 假網站冒充政府
     - 虛假購物平台
     - 愛情詐騙

## 快速開始

### 1. 啟動後端服務

```bash
cd backend
python main.py
```

確保看到以下輸出：
```
✅ AgentService 初始化完成
INFO:     Uvicorn running on http://127.0.0.1:8000
```

### 2. 打開前端界面

在瀏覽器中打開：
```
http://localhost:8000/personal_chat.html
```

或直接打開本地文件：
```
frontend/personal_chat.html
```

### 3. 選擇對話模式

點擊選擇您想要的模式：
- **防詐助手** - 獲取防詐建議
- **騙徒模擬** - 體驗詐騙手法（需選擇騙局類型）

### 4. 開始對話

點擊「開始對話」按鈕，開始與 AI 進行對話。

## 界面功能

### 聊天區域
- **自動滾動**：新消息自動滾動到底部
- **訊息氣泡**：不同角色使用不同顏色
  - 用戶：紫色漸變
  - 助手：綠色邊框
  - 騙徒：橙色邊框

### 工具按鈕
- **🗑️ 清除對話**：清除當前所有對話記錄
- **🔄 重新開始**：結束當前會話，返回模式選擇
- **📥 匯出對話**：將對話記錄匯出為文本文件

### 輸入框
- **Enter 發送**：按 Enter 鍵發送消息
- **Shift+Enter 換行**：按住 Shift + Enter 換行

## API 端點說明

### 1. 開始對話
```http
POST /api/personal-chat/start
Content-Type: application/json

{
    "mode": "assistant",  // 或 "scammer"
    "scam_type": "投資理財詐騙"  // 僅在 scammer 模式需要
}

Response:
{
    "session_id": "uuid",
    "reply": "歡迎消息",
    "mode": "assistant",
    "history": [],
    "success": true
}
```

### 2. 發送消息
```http
POST /api/personal-chat/message
Content-Type: application/json

{
    "session_id": "uuid",
    "message": "用戶消息"
}

Response:
{
    "session_id": "uuid",
    "reply": "AI 回應",
    "mode": "assistant",
    "history": [...],
    "success": true
}
```

### 3. 獲取會話信息
```http
GET /api/personal-chat/session/{session_id}

Response:
{
    "session_id": "uuid",
    "mode": "assistant",
    "history": [...],
    "scam_type": null
}
```

### 4. 結束會話
```http
DELETE /api/personal-chat/session/{session_id}

Response:
{
    "success": true,
    "message": "會話已結束"
}
```

### 5. 獲取可用模式
```http
GET /api/personal-chat/modes

Response:
{
    "modes": [
        {
            "id": "assistant",
            "name": "防詐助手",
            "description": "退休警察黃sir...",
            "icon": "🛡️"
        },
        {
            "id": "scammer",
            "name": "騙徒模擬",
            "description": "職業騙徒林志強...",
            "icon": "🎭"
        }
    ],
    "scam_types": [...]
}
```

## 技術架構

### 後端
- **框架**：FastAPI
- **AI 系統**：Google ADK + Ollama
- **模型**：
  - 助手：`anti-fraud-expert-20251112_095003`
  - 騙徒：`anti-fraud-scammer-20251112_095003`

### 前端
- **純 HTML/CSS/JavaScript**
- **無需框架**，直接可用
- **響應式設計**，支持手機和桌面

### 會話管理
- 使用內存存儲（生產環境建議使用數據庫）
- 每個會話獨立管理對話歷史
- 支持多個並發會話

| 特性 | 個人對話模式 | RPG 模式 | 自動模擬模式 |

## 注意事項

### ⚠️ 重要提醒

1. **騙徒模擬僅供學習**
   - 此功能僅用於教育目的
   - 體驗詐騙手法以提高防範意識
   - 請勿用於任何非法用途

2. **後端必須運行**
   - 使用前確保後端服務已啟動
   - 檢查 Ollama 服務是否運行
   - 確認模型已正確加載

3. **會話管理**
   - 會話存儲在內存中，服務重啟後會丟失
   - 生產環境建議使用持久化存儲

4. **性能考慮**
   - AI 回應需要數秒時間
   - 避免快速連續發送大量消息
   - 建議單次對話不超過 50 輪

## 常見問題

### Q1: 為什麼 AI 回應很慢？
A: AI 模型需要時間處理，通常需要 3-10 秒。如果超過 30 秒，請檢查：
- Ollama 服務是否正常運行
- 模型是否正確加載
- 網絡連接是否正常

### Q2: 可以同時開啟多個對話嗎？
A: 可以！每個瀏覽器標籤頁都是獨立的會話。

### Q3: 如何保存對話記錄？
A: 點擊「📥 匯出對話」按鈕，會自動下載文本文件。

### Q4: 騙徒模擬是否會真的詐騙我？
A: 不會！這是模擬環境，僅用於教育。系統不會要求真實個人信息或金錢。

### Q5: 如何切換到另一個模式？
A: 點擊「🔄 重新開始」按鈕，返回模式選擇界面。
