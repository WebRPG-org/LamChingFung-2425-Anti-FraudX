### 將RPG入口設為系統主頁

**之前**：

**現在**：

## 🔧 修改的文件

### 1. backend/api/frontend_routes.py

#### 根路徑路由
```python
@router.get("/")
async def serve_frontend():
    """提供RPG入口頁面作為主頁"""
    rpg_path = os.path.join(..., 'RPG_platform', 'RPG_Project', 'index.html')
    return FileResponse(rpg_path, headers=headers)
```

**改變**：
- 從指向 `frontend/index.html`
- 改為指向 `RPG_platform/RPG_Project/index.html`

---

#### 新增路由
```python
@router.get("/modes")
async def serve_modes_page():
    """提供原來的模式選擇頁面（如需要）"""
    modes_path = os.path.join(..., 'frontend', 'index.html')
    return FileResponse(modes_path, headers=headers)
```

**用途**：
- 如果還需要訪問原來的模式選擇頁面
- 可以通過 `/modes` 路徑訪問

---

### 2. RPG_platform/RPG_Project/index.html

#### 移除返回按鈕
```html
<!-- 之前 -->
<a href="http://localhost:8000/" class="back-button">🏠 返回主頁</a>

<!-- 現在 -->
<!-- 已移除，因為這個頁面就是主頁 -->
```

**原因**：
- RPG入口現在就是主頁
- 不需要返回按鈕
- 避免用戶混淆

## 💡 設計理念

### 為什麼這樣設置？

1. **RPG為核心**
   - 用戶主要使用RPG功能
   - 直接進入RPG入口更高效

2. **減少導航層次**
   - 少一步點擊
   - 更快到達目標

3. **保留原頁面**
   - 通過 `/modes` 仍可訪問
   - 如需要可隨時查看

4. **統一返回行為**
   - 所有頁面都返回RPG入口
   - 導航邏輯清晰

## 🔄 如果需要恢復

### 恢復原來的主頁設置

如果需要恢復模式選擇頁面為主頁：

```python

@router.get("/")
async def serve_frontend():
    """恢復原模式選擇頁面為主頁"""
    frontend_path = os.path.join(..., 'frontend', 'index.html')
    return FileResponse(frontend_path, headers=headers)
```

並在RPG入口頁面添加返回按鈕：
