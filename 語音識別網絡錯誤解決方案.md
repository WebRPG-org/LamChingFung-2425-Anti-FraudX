# 語音識別網絡錯誤解決方案

## ❌ 錯誤信息
```
語音識別失敗: network
```

## 🔍 問題原因

Web Speech API 的語音識別功能需要連接到Google的語音服務器，當出現 `network` 錯誤時，通常是以下原因：

1. **網絡連接問題**
2. **防火墻或安全軟件阻止**
3. **VPN干擾**
4. **瀏覽器限制**
5. **Google服務不可用**

---

## 🛠️ 解決方案

### 方案1：檢查網絡連接 ✅

#### 步驟1：測試網絡
```bash
# 打開瀏覽器控制台（F12）
# 輸入以下命令測試
fetch('https://www.google.com')
  .then(() => console.log('✅ 網絡正常'))
  .catch(() => console.log('❌ 網絡異常'));
```

#### 步驟2：檢查連接狀態
```javascript
// 檢查是否在線
console.log('在線狀態:', navigator.onLine);

// 如果顯示 false，請檢查：
// 1. Wi-Fi/網線是否連接
// 2. 路由器是否正常
// 3. 是否欠費
```

---

### 方案2：關閉VPN ⚠️

**如果您正在使用VPN**：

1. **臨時關閉VPN**
   - 某些VPN會阻止Google語音服務
   - 嘗試關閉VPN後重試

2. **更換VPN節點**
   - 選擇香港或台灣節點
   - 避免使用大陸節點

3. **VPN分流設置**
   - 將 `*.google.com` 設為直連
   - 不通過VPN訪問

---

### 方案5：檢查瀏覽器設置 ⚙️

#### Chrome/Edge設置

1. **檢查麥克風權限**
   ```
   地址欄輸入: chrome://settings/content/microphone
   
   確保：
   - 「網站可以要求使用麥克風」已啟用
   - http://localhost:8000 在「允許」列表中
   ```

2. **檢查安全連接**
   ```
   地址欄輸入: chrome://flags
   
   搜索: "insecure origins treated as secure"
   
   添加: http://localhost:8000
   ```

3. **清除瀏覽器數據**
   ```
   1. Ctrl+Shift+Delete
   2. 選擇「Cookie和其他網站數據」
   3. 選擇「緩存的圖片和文件」
   4. 點擊「清除數據」
   5. 重啟瀏覽器
   ```

### 方案6：使用HTTPS 🔒

Web Speech API 在非安全連接下可能受限。

#### 開發環境解決方案

**選項A：使用ngrok（推薦）**
```bash
# 安裝ngrok
# 訪問 https://ngrok.com/download

# 啟動後端
python backend/main.py

# 在新終端啟動ngrok
ngrok http 8000

# 使用ngrok提供的https鏈接訪問
# 例如：https://abc123.ngrok.io
```

**選項B：生成本地SSL證書**
```bash
# 安裝mkcert
# Windows: choco install mkcert
# Mac: brew install mkcert

# 生成證書
mkcert -install
mkcert localhost 127.0.0.1 ::1

# 修改後端啟動命令
uvicorn main:app --host 0.0.0.0 --port 8000 --ssl-keyfile=localhost-key.pem --ssl-certfile=localhost.pem
```

## 🧪 診斷工具

### 完整診斷腳本

在瀏覽器控制台（F12）運行：

```javascript
console.log('=== 語音識別診斷工具 ===\n');

// 1. 檢查瀏覽器支持
console.log('1. 瀏覽器支持:');
console.log('   SpeechRecognition:', 'webkitSpeechRecognition' in window || 'SpeechRecognition' in window);
console.log('   SpeechSynthesis:', 'speechSynthesis' in window);

// 2. 檢查網絡狀態
console.log('\n2. 網絡狀態:');
console.log('   在線狀態:', navigator.onLine);
console.log('   用戶代理:', navigator.userAgent);

// 3. 測試網絡連接
console.log('\n3. 測試Google連接...');
fetch('https://www.google.com', { mode: 'no-cors' })
  .then(() => console.log('   ✅ Google可訪問'))
  .catch(() => console.log('   ❌ Google不可訪問'));

// 4. 檢查麥克風權限
console.log('\n4. 檢查麥克風權限...');
navigator.mediaDevices.getUserMedia({ audio: true })
  .then(() => console.log('   ✅ 麥克風權限已授予'))
  .catch(err => console.log('   ❌ 麥克風權限錯誤:', err.message));

// 5. 測試語音識別
console.log('\n5. 測試語音識別...');
try {
    const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
    const recognition = new SpeechRecognition();
    recognition.lang = 'zh-HK';
    
    recognition.onerror = (event) => {
        console.log('   ❌ 語音識別錯誤:', event.error);
    };
    
    recognition.onstart = () => {
        console.log('   ✅ 語音識別啟動成功');
        recognition.stop();
    };
    
    recognition.start();
} catch (error) {
    console.log('   ❌ 無法創建語音識別:', error.message);
}

// 6. 檢查語音合成
console.log('\n6. 測試語音合成...');
if ('speechSynthesis' in window) {
    const voices = speechSynthesis.getVoices();
    console.log('   可用語音數量:', voices.length);
    const cantoneseVoice = voices.find(v => v.lang.includes('zh-HK') || v.lang.includes('yue'));
    console.log('   廣東話語音:', cantoneseVoice ? '✅ 有' : '❌ 無');
}

console.log('\n=== 診斷完成 ===');
console.log('請截圖以上結果，以便進一步排查問題。');
```

## 📊 錯誤碼對照表

| 錯誤碼 | 說明 | 解決方案 |
|--------|------|----------|
| **network** | 網絡連接失敗 | 檢查網絡、VPN、防火墻 |
| **not-allowed** | 權限被拒絕 | 允許麥克風訪問 |
| **no-speech** | 未檢測到語音 | 檢查麥克風、說話 |
| **audio-capture** | 音頻捕獲失敗 | 檢查麥克風硬件 |
| **aborted** | 被中止 | 重新嘗試 |
| **service-not-allowed** | 服務不允許 | 檢查瀏覽器設置 |

## 💬 常見問題

### Q: 為什麼語音識別需要網絡？
**A**: Web Speech API使用Google的雲端語音識別服務，需要將音頻發送到Google服務器進行處理。

### Q: 可以離線使用嗎？
**A**: Web Speech API不支持離線。如需離線功能，需要使用Google Cloud API或其他離線語音引擎。

### Q: VPN一定要關閉嗎？
**A**: 不一定。但某些VPN會阻止Google服務，建議：
- 選擇香港/台灣節點
- 設置分流規則
- 或臨時關閉測試

### Q: 在中國大陸可以使用嗎？
**A**: 需要VPN或代理訪問Google服務。建議：
- 使用穩定的VPN
- 選擇香港節點
- 或使用國內語音服務替代
