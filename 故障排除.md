# 🔧 故障排除指南

## ❗ 問題：ModuleNotFoundError: No module named 'google.adk'

### 原因
依賴未安裝或安裝失敗

### 解決方案

#### 方案 1：安裝所有依賴（推薦）

```bash
cd AI-Agent-main
安裝依賴.bat
```

或手動：
```bash
cd AI-Agent-main/backend
pip install -r requirements.txt
```

#### 方案 2：只安裝核心依賴（快速方案）

如果完整安裝失敗，可以只安裝核心依賴：

```bash
cd AI-Agent-main/backend
pip install fastapi uvicorn requests python-dotenv python-multipart ollama chromadb google-adk
```

#### 方案 3：使用快速啟動腳本

```bash
cd AI-Agent-main
快速啟動.bat
```

這個腳本會自動安裝核心依賴並啟動系統。

## ❗ 問題：Ollama 連接失敗

### 錯誤訊息
```
Connection refused
AI 服務錯誤: ...
```

### 解決方案

#### 1. 檢查 Ollama 是否運行

```bash
# 測試連接
curl http://localhost:11434/api/tags

# 如果失敗，啟動 Ollama
ollama serve
```

#### 2. 下載模型

```bash
# 下載 4B 模型（推薦）
ollama pull gemma3:4b

# 或下載 27B 模型
ollama pull gemma3:27b
```

#### 3. 檢查 .env 配置

確保 `backend/.env` 內容正確：
```
AGENT_MODEL=gemma3:4b
OLLAMA_BASE_URL=http://127.0.0.1:11434
```

## ❗ 問題：端口被占用

### 錯誤訊息
```
Address already in use
Port 8000 is already in use
```

### 解決方案

#### 1. 查找占用端口的程序

```powershell
# Windows
netstat -ano | findstr :8000
```

#### 2. 結束占用的進程

```powershell
# 找到 PID 後
taskkill /PID <PID> /F
```

#### 3. 或修改端口

編輯 `backend/main.py` 最後一行：
```python
uvicorn.run("main:app", host="127.0.0.1", port=8001, reload=True)
```

## ❗ 問題：.env 文件不存在

### 錯誤訊息
```
錯誤: .env 文件不存在
```

### 解決方案

在 `AI-Agent-main/backend/` 創建 `.env` 文件：

```
AGENT_MODEL=gemma3:4b
OLLAMA_BASE_URL=http://127.0.0.1:11434
```

或使用快速啟動腳本自動創建：
```bash
快速啟動.bat
```

## ❗ 問題：Python 版本問題

### 錯誤訊息
```
SyntaxError: ...
```

### 解決方案

確保使用 Python 3.10 或更高版本：

```bash
python --version
# 應該顯示 Python 3.10.x 或更高
```

如果版本過低，請升級 Python。

## ❗ 問題：RPG Maker 插件無法連接

### 錯誤訊息
```
網路連接失敗
無法連接到 AI 服務
```

### 解決方案

#### 1. 確認後台運行

訪問 http://localhost:8000，應該看到 Web UI

#### 2. 檢查插件配置

確保插件中的 URL 正確：
```javascript
const API_URL = 'http://localhost:8000/chat';
```

#### 3. 測試 API

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d "{\"role\":\"測試\",\"message\":\"你好\",\"history\":[]}"
```

#### 4. 查看 F8 控制台

在 RPG Maker 測試時按 F8，查看 Console 的錯誤訊息

## ❗ 問題：依賴安裝失敗

### 錯誤訊息
```
ERROR: Could not find a version that satisfies...
```

### 解決方案

#### 1. 升級 pip

```bash
python -m pip install --upgrade pip
```

#### 2. 分步安裝

```bash
# 先安裝 FastAPI 相關
pip install fastapi uvicorn

# 再安裝其他
pip install requests python-dotenv python-multipart

# 最後安裝 AI 相關
pip install ollama chromadb
```

#### 3. 使用國內鏡像（中國用戶）

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## ❗ 問題：WebSocket 連接失敗

### 錯誤訊息
```
WebSocket connection failed
```

### 解決方案

這是 AI-Agent 原有功能的問題，不影響 RPG Maker 使用。

RPG Maker 使用 HTTP API，不使用 WebSocket。

如果需要使用 Web UI：
1. 確保後台完全啟動
2. 刷新瀏覽器
3. 查看瀏覽器控制台錯誤

## 🆘 **仍然無法解決？**

### 最小化啟動方案

如果所有方法都失敗，嘗試最小化配置：

1. **只安裝核心依賴**
   ```bash
   pip install fastapi uvicorn requests python-dotenv
   ```

2. **創建簡化的 .env**
   ```
   AGENT_MODEL=gemma3:4b
   OLLAMA_BASE_URL=http://127.0.0.1:11434
   ```

3. **直接運行**
   ```bash
   cd backend
   uvicorn main:app --host 127.0.0.1 --port 8000
   ```

## 📞 **需要更多幫助？**

### 檢查日誌

後台啟動時會顯示詳細日誌，複製完整錯誤訊息進行分析。

### 常見錯誤模式

- `ModuleNotFoundError` → 依賴問題
- `Connection refused` → Ollama 問題
- `Address in use` → 端口問題
- `Permission denied` → 權限問題
- `SyntaxError` → Python 版本問題

---

**🔧 大部分問題都可以通過 `快速啟動.bat` 或 `安裝依賴.bat` 解決！**
