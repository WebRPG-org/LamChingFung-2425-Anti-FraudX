## 🎯 核心功能設計

### 四代理系統
```
ScammerAgent（騙徒）
├── 5種詐騙手法
├── 策略漸進性
├── 人格適應
└── 字數限制：50-100字

ExpertAgent（專家）
├── 防詐專家角色
├── 四種人格策略
├── 具體建議
└── 字數限制：80-100字

VictimAgent（受害者）
├── 4種人格
├── 情緒狀態變化
├── 自然反應
└── 字數限制：30-80字

RecorderAgent（記錄員）
├── 詳細分析
├── JSON輸出
├── 性能評分
└── 改進建議
```

### 信任度系統
```
三維度追蹤
├── trust_in_scammer (0-100)
├── trust_in_expert (0-100)
└── alertness (0-100)

動態修改器
├── 慣性修改器
├── 疲勞修改器
├── 情緒修改器
└── 人格修改器

結果判定
├── 騙徒贏：trust_in_scammer >= 80
├── 專家贏：trust_in_expert >= 75
└── 自我保護：alertness >= 80
```

### 並行回應生成
```
asyncio.gather()
├── 同時生成騙徒回應
├── 同時生成專家回應
└── 性能提升：~50%
```

### 會話持久化
```
前端（sessionStorage）
├── 保存session_id
├── 保存消息
├── 保存信任值
└── 自動恢復

後端（SQLite）
├── 會話記錄
├── 對話歷史
└── 性能統計
```

### 知識與評分系統
```
RAG知識庫
├── 281個真實案例
├── ChromaDB向量存儲
└── 按類型查詢

自適應評分
├── 基於人格的權重
├── 騙徒性能評分
└── 專家性能評分
```
