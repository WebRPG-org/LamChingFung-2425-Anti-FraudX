根據您的需求選擇不同的啟動方式：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【方式一】本地開發模式 (推薦用於開發和測試) ⭐
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 優點:
   • 啟動快速 (約10秒)
   • 代碼修改即時生效 (熱重載)
   • 不需要 Docker
   • 配置簡單

❌ 缺點:
   • 使用 CPU (性能略低)
   • 需要手動安裝 Python 環境

📝 啟動方式:
   雙擊運行: 本地启动.bat
   或手動: python start_server.py

🛑 停止服務:
   在終端按 Ctrl+C 或運行: 停止服务.bat

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【方式二】Docker 模式 (用於生產環境)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 優點:
   • 使用 GPU 加速 (更快)
   • 環境隔離
   • 適合生產部署

❌ 缺點:
   • 需要 Docker Desktop
   • 啟動較慢 (約30-60秒)
   • 需要額外配置

📝 啟動方式:
   1. 確保 Docker Desktop 正在運行
   2. 雙擊運行: 一键启动.bat

🛑 停止服務:
   docker-compose -f docker-compose.local.yml down

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【當前問題解決】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

您遇到的錯誤是因為:
   Docker Desktop 沒有運行，但 一键启动.bat 嘗試使用 Docker

解決方案:
   選項A: 使用本地模式 → 運行 本地启动.bat ✅
   選項B: 啟動 Docker Desktop 後再運行 一键启动.bat

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【快速測試】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

啟動後測試連接:
   • 運行: 測試連接.bat
   • 或運行: python test_ollama_connection.py
   • 或瀏覽器訪問: http://localhost:8000

【常用命令】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

本地模式:
   啟動: 本地启动.bat
   停止: 停止服务.bat 或 Ctrl+C
   測試: 測試連接.bat

Docker模式:
   啟動: 一键启动.bat
   停止: docker-compose -f docker-compose.local.yml down
   查看日誌: docker logs -f ai-antiscam-backend
   查看容器: docker ps

查看GPU使用:
   nvidia-smi
   nvidia-smi -l 1  (持續監控)
