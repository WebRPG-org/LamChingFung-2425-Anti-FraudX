# 人設與騙局輪換系統說明

## 修改摘要

已成功在 `RotatingScamSystem.js` 中實現**人設自動輪換**功能,現在系統會同時輪換:
1. ✅ **騙局類型** (10種騙局,循環輪換)
2. ✅ **受害人設** (4種人設,循環輪換)

## 系統架構

### 變數配置
| 變數ID | 用途 | 預設值 | 說明 |
|--------|------|--------|------|
| 11-16 | NPC騙局索引 | 1-6 | 每個NPC當前使用的騙局編號(1-10) |
| 20 | 對話總次數 | 0 | 統計累計對話次數 |
| 21 | 會話ID | - | 當前AI對話的session_id |
| 22 | 當前人設 | 'A' | 當前使用的人設類型(A/B/C/D) |
| 23 | 對話歷史 | [] | 當前對話的完整記錄 |
| 24 | 評分結果 | - | 上次對話的AI評分 |
| 25 | 人設索引 | 0 | 當前人設在PERSONAS數組中的索引 |

### 人設配置

```javascript
const PERSONAS = ['A', 'B', 'C', 'D'];

const PERSONA_NAMES = {
    'A': '長者',
    'B': '一般市民', 
    'C': '過度自信者',
    'D': '年輕學生'
};
```

### 10種騙局類型

1. **電話詐騙** - 假冒公安/法院
2. **投資詐騙** - 虛假高回報投資
3. **網購詐騙** - 假冒客服退款
4. **釣魚詐騙** - 偽造網站盜取資料
5. **愛情詐騙** - 感情欺騙索取金錢
6. **求職詐騙** - 虛假招聘收費
7. **中獎詐騙** - 假冒中獎通知
8. **貸款詐騙** - 虛假貸款騙取費用
9. **冒充親友** - 假冒熟人借錢
10. **虛假慈善** - 假借慈善募款

## 核心函數

### 1. `initializeSystem()`
```javascript
function initializeSystem() {
    // 初始化6個NPC的騙局
    for (let i = 0; i < NPC_IDS.length; i++) {
        const varId = 11 + i;
        if (!$gameVariables.value(varId)) {
            $gameVariables.setValue(varId, (i % 10) + 1);
        }
    }
    
    // 初始化人設輪換
    if (!$gameVariables.value(25)) {
        $gameVariables.setValue(25, 0);    // 人設索引
        $gameVariables.setValue(22, 'A');   // 當前人設
    }
}
```

### 2. `rotateToNextScam(npcId)`
切換指定NPC的騙局到下一個(1→2→...→10→1循環)

```javascript
function rotateToNextScam(npcId) {
    const index = NPC_IDS.indexOf(npcId);
    const varId = 11 + index;
    let currentScam = $gameVariables.value(varId) || (index + 1);
    
    // 切換到下一個騙局（1-10循環）
    currentScam = (currentScam % 10) + 1;
    $gameVariables.setValue(varId, currentScam);
}
```

### 3. `rotateToNextPersona()` ⭐ 新增
切換到下一個人設(A→B→C→D→A循環)

```javascript
function rotateToNextPersona() {
    let currentIndex = $gameVariables.value(25) || 0;
    
    // 切換到下一個人設（0-3循環）
    currentIndex = (currentIndex + 1) % PERSONAS.length;
    const newPersona = PERSONAS[currentIndex];
    
    $gameVariables.setValue(25, currentIndex);  // 更新索引
    $gameVariables.setValue(22, newPersona);     // 更新人設
    
    return newPersona;
}
```

## 使用流程

### 第1次對話
```
初始狀態:
- NPC1: 騙局1 (電話詐騙)
- 人設: A (長者)

對話流程:
1. 玩家與NPC1對話
2. 顯示: "您的人設：長者"
3. 進行對話訓練
4. 結束後評分

輪換結果:
- NPC1騙局: 1 → 2 (投資詐騙)
- 人設: A → B (一般市民)
- 顯示: "NPC1 下次將使用：投資詐騙"
- 顯示: "下次人設：一般市民"
```

### 第2次對話
```
當前狀態:
- NPC1: 騙局2 (投資詐騙)
- 人設: B (一般市民)

對話流程:
1. 玩家再次與NPC1對話
2. 顯示: "您的人設：一般市民"
3. 進行對話訓練
4. 結束後評分

輪換結果:
- NPC1騙局: 2 → 3 (網購詐騙)
- 人設: B → C (過度自信者)
```

### 循環機制
```
騙局循環: 1 → 2 → 3 → ... → 10 → 1 → ...
人設循環: A → B → C → D → A → ...
```
