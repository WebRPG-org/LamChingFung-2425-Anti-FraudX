# 混合模式視覺分析說明

## 🎯 設計理念

**問題**：
- `gemma3:4b` (base model) 支持視覺 ✅，但沒有防詐騙訓練 ❌
- `anti-fraud-expert-20251117_173453` (fine-tuned) 有防詐騙訓練 ✅，但不支持視覺 ❌

**解決方案**：混合模式
- **階段 1**：用 `gemma3:4b` 理解圖片內容
- **階段 2**：用 `anti-fraud-expert` 生成專業回應

---

## 🔧 工作流程

### 階段 1: 圖片內容提取

**模型**：`gemma3:4b` (vision_model)

**任務**：理解圖片，提取所有資訊

**提示詞**：
```
請仔細閱讀圖片，提取以下資訊：

1. 圖片類型：WhatsApp、短訊、網站截圖等
2. 訊息內容：逐字引用所有文字
3. 可疑元素：網址、電話、時間限制、要求資料、聲稱官方
4. 視覺特徵：Logo、語言、排版風格
```

**輸出示例**：
```
1. 圖片類型：WhatsApp 訊息截圖

2. 訊息內容：
「系統檢測你的账号在风险，请在12小时内登录安全中心 t.ly/WS-RTD 进行确认，将限时使用。」

3. 可疑元素：
- 網址連結：t.ly/WS-RTD（短網址）
- 時間限制：12小时内
- 聲稱官方：「系統檢測」「安全中心」

4. 視覺特徵：
- 語言：簡體中文
- 平台：WhatsApp
- 排版：標準訊息格式
```

---

### 階段 2: 專家回應生成

**模型**：`anti-fraud-expert-20251117_173453` (expert_model)

**任務**：根據圖片內容，生成專業防詐騙建議

**輸入**：
- 階段 1 提取的圖片內容
- ExpertAgent 的系統提示詞（包含格式要求）
- 用戶問題

**輸出示例**：
```
🚨 呢個係詐騙！唔好點擊！

📱 圖片類型：WhatsApp 訊息

📝 訊息內容：
「系統檢測你的账号在风险，请在12小时内登录...」

⚠️ 點解係詐騙：
1. 逼你 12 小時內做嘢
2. 用短網址 t.ly/WS-RTD
3. 假扮官方安全中心

💡 你要做咩：
1. 立即刪除呢個訊息
2. 唔好點擊任何連結
3. 封鎖呢個號碼
```

## 📁 文件結構

### `backend/services/vision_service.py`

```python
class VisionService:
    def __init__(self, vision_model=None, expert_model=None):
        # 從 .env 讀取配置
        self.vision_model = vision_model or os.getenv("VISION_MODEL", "gemma3:4b")
        self.expert_model = expert_model or os.getenv("AGENT_MODEL_EXPERT", "anti-fraud-expert-20251117_173453")
    
    async def analyze_with_vision(self, message, images, system_prompt):
        # 階段 1: 提取圖片內容
        image_description = await self._extract_image_content(images)
        
        # 階段 2: 生成專家回應
        expert_response = await self._generate_expert_response(
            image_description, message, system_prompt
        )
        
        return expert_response
    
    async def _extract_image_content(self, images):
        # 使用 vision_model (gemma3:4b)
        response = ollama.chat(
            model=self.vision_model,
            messages=[{
                'role': 'user',
                'content': extraction_prompt,
                'images': images
            }]
        )
        return response['message']['content']
    
    async def _generate_expert_response(self, image_description, user_message, system_prompt):
        # 使用 expert_model (anti-fraud-expert)
        response = ollama.chat(
            model=self.expert_model,
            messages=[
                {'role': 'system', 'content': system_prompt},
                {'role': 'user', 'content': f"圖片內容：{image_description}\n用戶問題：{user_message}"}
            ]
        )
        return response['message']['content']
```

## 💡 為什麼這樣設計？

### 1. 分離關注點

**階段 1（視覺理解）**：
- 專注於「看懂」圖片
- 不需要生成完美的回應
- 只需要提取所有資訊

**階段 2（專業回應）**：
- 專注於「分析」和「建議」
- 使用 fine-tuned 模型的專業知識
- 遵守 ExpertAgent 的格式要求

### 2. 發揮各自優勢

**gemma3:4b**：
- ✅ 視覺能力強
- ✅ 可以讀取圖片中的文字
- ❌ 沒有防詐騙訓練

**anti-fraud-expert**：
- ✅ 防詐騙知識豐富
- ✅ 回應格式穩定
- ❌ 不支持視覺

**混合模式**：
- ✅ 結合兩者優勢
- ✅ 避免各自缺點

### 3. 靈活配置

**統一使用 `.env` 控制**：
```bash
# 可以輕鬆切換模型
VISION_MODEL=gemma3:4b
AGENT_MODEL_EXPERT=anti-fraud-expert-20251117_173453

# 或者使用其他模型
VISION_MODEL=llava:latest
AGENT_MODEL_EXPERT=gemma3:4b
```
