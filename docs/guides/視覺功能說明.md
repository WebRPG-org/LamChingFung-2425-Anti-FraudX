# Gemma 3 4B 視覺功能說明

## ✅ 確認：支持視覺功能

```bash
ollama show gemma3:4b
```

**輸出**：
```
Capabilities
  completion
  vision          ← ✅ 支持視覺！
```

**結論**：**Gemma 3 4B 是多模態模型，可以直接理解圖片內容！**

---

## 🎯 功能說明

### 可以做到
- ✅ **OCR 文字識別** - 讀取圖片中的文字
- ✅ **視覺理解** - 識別物體、場景、UI 元素
- ✅ **Logo 識別** - 識別品牌和標誌
- ✅ **內容分析** - 理解圖片整體內容
- ✅ **詐騙檢測** - 識別可疑特徵

### 實際應用
1. **識別詐騙短信截圖** - 自動讀取並分析短信內容
2. **檢查假網站** - 識別釣魚網站特徵
3. **分析投資詐騙** - 檢測虛假投資平台
4. **驗證官方通知** - 判斷是否為官方渠道

## 💡 使用方法

### Python 代碼示例

```python
import ollama
import base64

# 讀取圖片
with open('image.jpg', 'rb') as f:
    image_data = base64.b64encode(f.read()).decode('utf-8')

# 分析圖片
response = ollama.chat(
    model='gemma3:4b',
    messages=[{
        'role': 'user',
        'content': '請分析這張圖片，判斷是否為詐騙。',
        'images': [image_data]  # base64 編碼的圖片
    }]
)

print(response['message']['content'])
```

### 詐騙檢測提示詞

```python
prompt = """請分析這張圖片，判斷是否為詐騙。

請檢查以下特徵：
1. 是否有可疑的文字內容（如「立即」「緊急」「中獎」「轉帳」等）
2. 是否有可疑的連結或網址
3. 是否假冒官方機構或品牌
4. 是否要求提供個人資料或密碼
5. 是否製造緊急感或恐嚇

請用繁體中文回答，並給出：
- 風險等級（高/中/低）
- 詐騙特徵
- 建議措施"""
```

## 🎯 實際場景示例

### 場景 1: 詐騙短信截圖

**用戶操作**：
1. 收到可疑短信
2. 截圖
3. 上傳到系統
4. 詢問：「這個短信是詐騙嗎？」

**AI 回應**：
```

📝 短信內容：
「【銀行通知】您的帳戶出現異常活動，請立即點擊 
hxxps://fake-bank.com/verify 驗證身份，否則將被凍結。」

🚨 這是典型的釣魚詐騙短信！

詐騙特徵：
⚠️ 假冒銀行官方
⚠️ 製造緊急感（「立即」「異常」「凍結」）
⚠️ 可疑連結（fake-bank.com 不是官方域名）
⚠️ 要求驗證身份

風險等級：高

建議措施：
✅ 不要點擊任何連結
✅ 直接致電銀行官方熱線（2123 4567）
✅ 刪除此短信
✅ 向警方舉報（18222）
```

## ⚠️ 注意事項

### 模型限制
1. **模型大小**：4B 參數相對較小，可能無法處理極複雜場景
2. **語言支持**：對繁體中文 OCR 的支持可能有限
3. **準確性**：不是 100% 準確，需要人工確認

### 使用建議
- ✅ 用於初步篩選和快速判斷
