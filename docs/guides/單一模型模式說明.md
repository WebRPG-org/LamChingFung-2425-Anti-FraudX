# 單一模型模式說明

## 🎯 架構變更

從**混合模式**改回**單一模型模式**。

---

## 🔄 架構對比

### 之前：混合模式（兩階段）

```
用戶上傳圖片
    ↓
階段 1: gemma3:4b 提取圖片內容
    ↓
階段 2: anti-fraud-expert 生成回應
    ↓
返回給用戶
```

**問題**：
- ❌ anti-fraud-expert 不遵守格式
- ❌ 回應不相關（「我會分析」）
- ❌ 兩個模型協調困難

---

### 現在：單一模型模式

```
用戶上傳圖片
    ↓
gemma3:4b 直接分析並生成回應
    ↓
返回給用戶
```

**優點**：
- ✅ 簡單直接
- ✅ 更容易控制格式
- ✅ 更快的響應速度
- ✅ 更好的指令遵循

---

## 🔧 代碼變更

### `vision_service.py`

**之前**：
```python
class VisionService:
    def __init__(self, vision_model: str = None, expert_model: str = None):
        self.vision_model = "gemma3:4b"
        self.expert_model = "anti-fraud-expert-20251117_173453"
    
    async def analyze_with_vision(...):
        # 階段 1: 提取圖片內容
        image_description = await self._extract_image_content(images)
        
        # 階段 2: 生成專家回應
        expert_response = await self._generate_expert_response(...)
        
        return expert_response
```

**現在**：
```python
class VisionService:
    def __init__(self, vision_model: str = None):
        self.vision_model = "gemma3:4b"
    
    async def analyze_with_vision(...):
        # 直接分析並生成回應
        messages = [{'role': 'user', 'content': analysis_prompt, 'images': images}]
        response = ollama.chat(model=self.vision_model, messages=messages)
        
        return response['message']['content']
```

## 🔄 工作流程

### 完整流程

```
1. 用戶上傳圖片
   ↓
2. 前端轉換為 base64
   ↓
3. 後端接收（personal_chat_routes.py）
   ↓
4. 調用 VisionService.analyze_with_vision()
   ↓
5. 構建提示詞（包含格式要求）
   ↓
6. 調用 ollama.chat(model="gemma3:4b", images=...)
   ↓
7. gemma3:4b 讀取圖片 + 分析 + 生成回應
   ↓
8. 返回格式化的回應
   ↓
9. 顯示給用戶
```
