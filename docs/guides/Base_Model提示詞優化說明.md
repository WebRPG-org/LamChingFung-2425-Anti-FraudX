**問題**：
- ❌ 完全沒有按照格式
- ❌ 充滿星號和重複文字
- ❌ 內容混亂，不連貫
- ❌ 沒有使用指定的 emoji
- ❌ 沒有遵守字數限制

---

## 💡 根本原因

### Base Model 的限制

**問題**：`gemma3:4b` base model 沒有經過 fine-tuning，對複雜的提示詞反應不佳。

**表現**：
- 難以理解複雜的格式要求
- 容易產生重複內容
- 不遵守字數限制
- 過度使用星號和 emoji

---

## ✅ 解決方案

### 方案 1: 簡化提示詞（已實施）

**策略**：將格式要求直接放在用戶消息中，而不是系統提示詞。

#### 修改前（複雜）

```python
system_prompt = """
🌐 **語言要求（最高優先級）**：
- ✅ **必須使用繁體中文**
- ✅ **必須使用香港廣東話口語**
...（很多規則）

📝 **格式要求（必須遵守）**：
- ✅ **嚴格遵守系統提示詞的格式**
...（很多規則）
"""
```

#### 修改後（簡化）

```python
# 系統提示詞（簡短）
system_prompt = """
你係一個香港防詐騙專家。用繁體中文廣東話回應。

**回應格式（必須完全照做）**：
[格式範例]

**重要規則**：
- 只用 5 個 emoji
- 只講 3 個重點
- 每點最多 15 字
"""

# 用戶消息（包含格式）
user_message = """
請用以下格式回應（用繁體中文廣東話）：

🚨 呢個係詐騙！唔好點擊！
📱 圖片類型：[類型]
...

用戶問題：[實際問題]
"""
```

**優點**：
- ✅ Base model 更容易理解
- ✅ 格式要求更明確
- ✅ 減少混亂的回應

```python
def clean_response(response):
    # 移除過多星號
    response = re.sub(r'\*{3,}', '**', response)
    # 移除重複句子
    lines = response.split('\n')
    unique_lines = []
    for line in lines:
        if line not in unique_lines:
            unique_lines.append(line)
    return '\n'.join(unique_lines)
```
