# 騙徒戰術訓練系統說明

## 📋 概述

本系統從 `scraped_alerts.json` 中的真實防騙警示提取騙徒手法，生成按**騙局類型分類**的訓練數據，讓騙徒模型學習各種詐騙戰術。

## 🎯 核心特點

### 1. 按騙局類型訓練
騙徒會根據不同的騙局類型使用相應的手法：

| 騙局類型 | 身份 | 核心手法 |
|---------|------|---------|
| **假冒官員** | 政府官員/執法人員 | 權威施壓、製造恐慌、要求配合調查、轉駁內地公安 |
| **假冒銀行** | 銀行職員 | 假網站連結、索取帳戶密碼、製造帳戶危機 |
| **虛假投資** | 投資顧問/導師 | 高回報承諾、先給甜頭、內部消息、保證獲利 |
| **釣魚詐騙** | 官方機構/公司 | 假連結、假網站、索取個人資料、惡意程式 |
| **中獎詐騙** | 活動主辦方/客服 | 製造貪念、限時優惠、要求先繳費 |
| **求職詐騙** | 僱主/招聘人員 | 高薪誘惑、在家工作、收取費用、洗黑錢陷阱 |
| **WhatsApp詐騙** | 親友/熟人 | 假冒親友、緊急求助、拒絕通話 |
| **愛情詐騙** | 網上情人 | 建立感情、長期培養、製造困境、要求金錢 |

### 2. 四階段話術訓練

每種騙局類型都包含四個階段的訓練樣本：

#### 階段 1: 開場話術 (Opening)
- **權威型開場**：假冒官員/銀行
- **利誘型開場**：投資/中獎
- **親切型開場**：求職/愛情

#### 階段 2: 施壓話術 (Pressure)
- **時間壓力**：「只剩30分鐘」「馬上過期」
- **恐嚇威脅**：「會被拘捕」「帳戶凍結」
- **損失恐懼**：「錯過機會」「名額有限」

#### 階段 3: 索取資料/金錢 (Extraction)
- **索取銀行資料**：帳號、密碼、驗證碼
- **要求轉帳**：「安全帳戶」「監管帳戶」
- **要求先付費**：手續費、稅款、保證金

#### 階段 4: 對抗專家 (Counter Expert)
- **攻擊專家可信性**：「佢哋唔了解你嘅情況」
- **製造對立**：「你信我定信佢哋？」
- **加強緊迫感**：「而家冇時間聽佢哋講」

### 3. 真實案例學習

每個訓練樣本都基於 `scraped_alerts.json` 中的真實案例，包含：
- 真實騙案的手法描述
- 警方呼籲的防範措施
- 案例的日期和來源

## 🚀 使用方法

### 方法 1: 使用批次腳本（推薦）

```batch
騙徒戰術訓練.bat
```

選單選項：
- **[1] 生成騙徒戰術訓練數據** - 從 scraped_alerts.json 提取手法
- **[2] 訓練騙徒模型** - 使用生成的數據訓練
- **[3] 完整流程** - 生成 + 訓練一次完成
- **[4] 查看現有訓練數據** - 檢查已生成的文件

### 方法 2: 命令行執行

#### 步驟 1: 生成訓練數據
```bash
python backend/scripts/generate_scammer_tactics_from_alerts.py
```

可選參數：
- `--alerts-file` - 警示文件路徑（默認: data/scraped_alerts.json）
- `--output-dir` - 輸出目錄（默認: backend/training_data/finetuning）
- `--split-ratio` - 訓練集比例（默認: 0.9）

#### 步驟 2: 訓練模型
```bash
python backend/scripts/run_finetuning.py --model scammer
```

## 🔧 與現有系統整合

### 1. 與 scammer.py 整合

生成的訓練數據會增強 `backend/agents/scammer.py` 中的騙徒模型：

```python
class ScammerAgent(Agent):
    def __init__(self):
        local_model_name = os.getenv("AGENT_MODEL_SCAMMER") or "gemma3:4b"
        # 訓練後的模型會自動使用學到的手法
```

### 2. 與策略管理器整合

配合 `backend/utils/scammer_strategy_manager.py` 使用：

```python
# 策略管理器會根據騙局類型推薦策略
strategy = recommend_scammer_strategy(
    victim_trust_scammer=70,
    victim_trust_expert=40,
    victim_persona="elderly",
    scam_type="假冒官員"  # 新增：騙局類型
)
```

### 3. 與模擬系統整合

在 `backend/api/simulation_routes.py` 中使用：

```python
# 模擬時指定騙局類型
simulation_context = {
    "scam_type": "假冒銀行",
    "victim_persona": "average"
}
```

## 📈 訓練建議

### 1. 訓練頻率
- **初次訓練**：使用所有生成的數據
- **更新訓練**：當 scraped_alerts.json 更新時重新生成

### 2. 訓練參數
```bash
# 推薦參數
temperature: 0.8      # 保持創造性
top_p: 0.9           # 多樣性
top_k: 40            # 詞彙選擇
repeat_penalty: 1.1  # 避免重複
```

### 3. 評估標準
- ✅ 是否能識別並使用對應騙局類型的手法
- ✅ 是否能按階段推進（開場→施壓→索取→對抗）
- ✅ 是否能根據受害者反應調整策略
- ✅ 是否能有效對抗專家介入

## ⚠️ 重要提醒

### 倫理考量
1. **僅用於訓練**：此系統僅用於反詐騙訓練，不得用於實際詐騙
2. **明確標示**：所有使用場景都應明確標示為「模擬訓練」
3. **受控環境**：僅在受控的訓練環境中使用
4. **監督使用**：使用時應有適當的監督和審查機制

### 安全措施
- ✅ 系統提示中包含「⚠️ 警告：此為反詐騙訓練用模擬角色 ⚠️」
- ✅ 不提供真實的官方熱線號碼
- ✅ 避免提供可查證的真實資料
- ✅ 所有生成的假網址使用 `hxxps://` 格式（無法點擊）
