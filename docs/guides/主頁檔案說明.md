# 主頁檔案說明

## 🔍 問題發現

項目中存在**兩個主頁文件**，功能和設計都不同：

---

## 📋 兩個主頁對比

### 1. **frontend/index.html** ❌ 舊版主頁

**特點**：
- 📄 文件大小：37,168 bytes
- 🎨 設計：白色容器 + 詳細控制面板
- 🎯 功能：完整的訓練系統控制介面
- 📊 內容：參數選擇、狀態面板、對話顯示

**標題**：
```
🛡️ AI 反詐騙對話平台
多智能體真實 AI 對話
```

**主要功能**：
- ⚙️ 對話參數選擇（詐騙手法、受騙者類型、模式）
- 🔄 開始對話按鈕
- ⚡ 循環訓練按鈕
- 🛑 停止訓練按鈕
- 🗂 查看訓練記錄按鈕
- 📈 對話狀態面板
- 💬 實時對話顯示

**問題**：
- 這是一個**測試/訓練頁面**，不應該作為主頁
- 功能過於複雜，不適合作為首頁
- 缺少導航到其他模式的入口

---

### 2. **frontend/home.html** ✅ 新版主頁

**特點**：
- 📄 文件大小：14,918 bytes（更輕量）
- 🎨 設計：現代化卡片式佈局
- 🎯 功能：模式選擇入口頁面
- 📊 內容：兩張大型模式卡片

**標題**：
```
🛡️ AI 防詐騙訓練系統
多模式智能防詐教育平台
```

**主要功能**：
兩張精美的模式卡片：

#### 卡片 1：RPG 遊戲模式 🎮
```
互動式RPG遊戲
與NPC對話學習防詐技巧

特色：
• 支持人設和騙局輪換
• 遊戲化學習體驗
• 沉浸式防詐訓練
```

#### 卡片 2：個人對話模式 💬
```
一對一真實對話體驗
選擇與助手或騙徒交談

特色：
• 真實對話體驗
• 角色扮演學習
• 靈活對話模式
```

**優點**：
- ✅ 清晰的導航結構
- ✅ 現代化UI設計
- ✅ 適合作為系統入口
- ✅ 用戶友好的模式選擇

---

## 🎯 應該使用哪一個？

### ✅ 推薦：使用 `home.html` 作為主頁

**理由**：
1. **定位明確**：入口頁面，用於模式選擇
2. **設計現代**：卡片式佈局，視覺美觀
3. **導航清晰**：清楚展示兩個主要功能模式
4. **用戶友好**：簡潔明了，易於理解

---

### ❌ 不推薦：使用 `index.html` 作為主頁

**理由**：
1. **功能定位錯誤**：這是訓練控制頁面，不是主頁
2. **過於複雜**：參數、按鈕太多，不適合首頁
3. **缺少導航**：沒有到其他模式的入口
4. **設計陳舊**：白色容器設計較傳統

---

## 🔧 建議的解決方案

### 方案 1：重命名文件 ⭐⭐⭐⭐⭐ 推薦

```bash
# 1. 將舊的 index.html 重命名為功能性名稱
frontend/index.html → frontend/training_control.html

# 2. 將新的 home.html 重命名為 index.html
frontend/home.html → frontend/index.html
```

**優點**：
- ✅ 符合Web慣例（index.html = 主頁）
- ✅ 保留舊功能（改名為training_control.html）
- ✅ 清晰的文件命名

---

### 方案 2：修改後端路由 ⭐⭐⭐

在後端設置默認路由指向 `home.html`：

```python
# backend/main.py
@app.get("/")
async def root():
    return FileResponse("frontend/home.html")
```

**優點**：
- ✅ 不需要重命名文件
- ✅ 保持原有文件結構

**缺點**：
- ❌ 文件命名與用途不符
- ❌ 可能造成混淆

---

### 方案 3：刪除舊文件 ⭐⭐

如果 `index.html` 的功能已被其他頁面替代：

```bash
# 刪除舊的 index.html
rm frontend/index.html

# 將 home.html 重命名為 index.html
mv frontend/home.html frontend/index.html
```

**注意**：刪除前請確認功能已完全遷移！
