# 自動訓練模式說明 (Auto Training Mode)

## ✅ 確認:兩種模式都使用動態AI生成

### 📌 系統架構
- **共用插件**: `RotatingScamSystem.js` 同時支持手動和自動模式
- **AI生成**: 兩種模式都使用 `/api/game/message` API 生成對話
- **輪換系統**: 騙局 (1-10循環) 和人設 (A→B→C→D→A) 在兩種模式都有效

---

## 🚀 自動訓練模式特性

### 1. 啟動方式
```
1. 與"訓練助理" NPC對話
2. 選擇"啟動訓練"
3. 選擇"自動訓練"
4. 系統開始自動運行
```

### 2. 工作流程

#### 自動循環:
```javascript
startAutoMode() 
  → autoModeActive = true
  → moveToNextNpc() // 移動到下一個NPC
      → 角色自動移動到NPC位置
      → startAIBattle(npcId, scam, true) // isAutoMode=true
          → startDialogueLoop() // 開始對話
              → generateVictimResponse() // 使用AI-A生成受害者回應
              → generateScammerResponse() // 使用AI-D生成騙徒回應
              → 重複8輪
          → endBattle() // 結束對話
              → 評分
              → rotateToNextScam() // 切換騙局
              → rotateToNextPersona() // 切換人設
      → 移動到下一個NPC (循環6個)
```

#### NPC位置 (Map002):
1. **NPC 1** (Event 12): 座標 (14, 11) - 投資騙局
2. **NPC 2** (Event 13): 座標 (19, 11) - 求職騙局  
3. **NPC 3** (Event 14): 座標 (14, 16) - 網戀騙局
4. **NPC 4** (Event 15): 座標 (19, 16) - 冒充公檢法
5. **NPC 5** (Event 16): 座標 (14, 21) - 網購騙局
6. **NPC 6** (Event 17): 座標 (19, 21) - 中獎騙局

### 3. 停止方式
- 再次與"訓練助理"對話
- 選擇"停止自動訓練"
- 或關閉遊戲

## 🔧 技術實現細節

### AI對話生成 (兩種模式共用)

#### 受害者回應 (`generateVictimResponse`)
```javascript
// 動態生成prompt
const prompt = `【人設背景】${personaName}(${personaDesc})
【當前騙局】第${scamNum}種:${scamName}
【對話歷史】${history}
【騙徒最新話術】${lastScammerMessage}
【要求】作為受害者自然回應,50-80字,廣東話口語`;

// 調用AI API
fetch('http://localhost:8000/api/game/message', {
    session_id: sessionId,
    message: prompt,
    target_ai: 'AI-A' // 受害者AI
});
```

#### 騙徒話術 (`generateScammerResponse`)
```javascript
// 動態生成prompt
const prompt = `【專業騙徒】使用${scamName}騙局
【目標人設】${personaName}(${personaDesc})
【戰術】${tacticDesc}
【要求】專業話術,60-100字,不暴露身份`;

// 調用AI API  
fetch('http://localhost:8000/api/game/message', {
    session_id: sessionId,
    message: prompt,
    target_ai: 'AI-D' // 騙徒AI
});
```

### 輪換系統

#### 騙局輪換 (10種循環)
```javascript
function rotateToNextScam(npcId) {
    const varId = 10 + npcId; // Variables 11-16
    let current = $gameVariables.value(varId);
    let next = (current % 10) + 1; // 1→2→...→10→1
    $gameVariables.setValue(varId, next);
}
```

#### 人設輪換 (4種循環)
```javascript
function rotateToNextPersona() {
    let currentIndex = $gameVariables.value(25); // 0-3
    let nextIndex = (currentIndex + 1) % 4; // 0→1→2→3→0
    $gameVariables.setValue(25, nextIndex);
    
    const personas = ['A', 'B', 'C', 'D'];
    $gameVariables.setValue(22, personas[nextIndex]);
}
```

## 📊 數據記錄

### 遊戲變數 (Game Variables)

| 變數ID | 用途 | 範圍 |
|--------|------|------|
| 11-16 | 各NPC當前騙局 | 1-10 |
| 20 | 當前NPC ID | 1-6 |
| 21 | 當前騙局編號 | 1-10 |
| 22 | 當前人設 | A/B/C/D |
| 23 | Session ID | string |
| 25 | 人設索引 | 0-3 |

### 後端數據庫記錄

每次對話都會保存到 `anti_fraud_game.db`:

```sql
-- sessions 表
CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    persona_type TEXT,    -- 當前人設 (A/B/C/D)
    scam_type TEXT,       -- 當前騙局 (1-10)
    created_at TIMESTAMP,
    last_activity TIMESTAMP
);

-- conversations 表  
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY,
    session_id TEXT,
    role TEXT,            -- 'scammer' or 'victim'
    message TEXT,         -- AI生成的對話內容
    timestamp TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id)
);
```

### 後端配置
- **URL**: `http://localhost:8000`
- **AI模型**: Ollama gemma:2b
- **API端點**:
  - `POST /api/game/start` - 創建會話
  - `POST /api/game/message` - AI生成對話
  - `GET /api/game/history/{session_id}` - 獲取對話歷史

## 🐛 故障排除

### 問題1: 自動模式不切換人設
**原因**: 可能是變數未初始化  
**解決**: 檢查變數25是否為0-3範圍內

### 問題2: AI生成失敗
**原因**: 後端API未啟動或Ollama未運行  
**解決**: 
```bash
# 檢查後端
curl http://localhost:8000/health

# 檢查Ollama
curl http://localhost:11434/api/tags
```

### 問題3: 自動模式卡住
**原因**: 移動路徑被阻擋  
**解決**: 確保NPC周圍沒有障礙物,或手動停止後重新啟動
