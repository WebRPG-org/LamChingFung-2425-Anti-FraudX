# 信任度系統調整報告

## 🔍 問題診斷

### 原始問題

從日誌中發現：

```
📊 受害者分析: 信任变化={'scammer': 5}, 情绪=anxious
📊 信任度更新 - 对骗徒: 100, 对专家: 50
```

**信任度從 75 直接跳到 100**（+25），導致：
- ❌ 第2輪就達到 100/100，模擬立即結束
- ❌ 沒有足夠的對話輪次（只有1-2輪）
- ❌ 訓練數據不足（對話太短）
- ❌ 不夠真實（實際詐騙需要多輪互動才能建立信任）

---

## 📊 問題根源

### elderly 類型的信任度變化規則（修改前）

```python
- 對方展示「官方身份」：+10
- 對方提供「著數」「福利」：+15
- 對方施加壓力「緊急」：+5
```

**計算**:
- 騙徒第一輪使用「authority + urgency + benefits」
- 信任度增加：+10 + +15 + +5 = **+30**
- 從初始 70 → 100，只需 1 輪！

### average 類型的信任度變化規則（修改前）

```python
- 對方展示專業證書/公司背景：+15
- 對方用專業術語解釋：+10
- 對方施加時間壓力：+10
```

**計算**:
- 騙徒第一輪使用多種策略
- 信任度增加：+15 + +10 + +10 = **+35**
- 從初始 60 → 95，只需 1 輪！

## 💡 設計理念

### 1. 漸進式信任建立

**真實詐騙過程**:
```
初次接觸 → 建立身份 → 製造情境 → 增加壓力 → 要求行動
   (+5)      (+5)        (+8)        (+5)       最終決定
```

**符合心理學原理**:
- 信任需要時間累積
- 人不會立即相信陌生人
- 需要多次交互才能建立關係

### 2. 平衡難度

**elderly 型** (更易受騙):
- 初始信任度較高（70）
- 每輪變化較快（±12）
- 閾值較低（>85）

**average 型** (較有警覺):
- 初始信任度中等（60）
- 每輪變化中等（±15）
- 閾值較高（>80）

**overconfident 型** (最警覺):
- 初始信任度較低（40）
- 需要更強的證據才會信任
- 閾值最高

### 3. 每輪最大變化限制

**為什麼需要限制？**
- ❌ 避免單輪暴增（+35）
- ✅ 確保至少 2-3 輪對話
- ✅ 給專家介入的時間窗口

**限制值設計**:
- elderly: ±12（較易波動）
- average: ±15（中等波動）
- 這是基於單個策略的平均值
