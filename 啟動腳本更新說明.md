## 🔧 自定義配置

### 修改默認模型

如果你想使用其他模型，可以編輯啟動腳本：

**本地启动.bat**:
```batch
# 找到這行（約第 42 行）
echo [步骤3/4] 檢查 Gemma 3 4B 模型...
ollama list | findstr "gemma3:4b" >nul 2>&1

# 改為
echo [步骤3/4] 檢查 Llama 3.2 3B 模型...
ollama list | findstr "llama3.2:3b" >nul 2>&1
```

**快速啟動.bat**:
```batch
# 找到這行（約第 29 行）
echo AGENT_MODEL=gemma3:4b

# 改為
echo AGENT_MODEL=llama3.2:3b
```

### 添加環境變量

可以在啟動腳本中添加自定義環境變量：

```batch
REM 在啟動 Python 服務器之前添加
set CUSTOM_VAR=value
set ANOTHER_VAR=another_value

python start_server.py
```

## ⚠️ 常見問題

### Q1: 啟動時提示 "Gemma 3 4B 未安裝"

**解決方法 1** (推薦): 在啟動腳本中選擇 `Y` 自動下載

**解決方法 2**: 手動下載
```bash
ollama pull gemma3:4b
```

### Q2: 下載 Gemma 3 4B 失敗

**可能原因**:
- 網絡連接問題
- 磁盤空間不足（需要 3.3GB）
- Ollama 服務未運行

**解決**:
```bash
# 1. 檢查網絡
ping ollama.com

# 2. 檢查磁盤空間
dir C:\

# 3. 重新啟動 Ollama
taskkill /F /IM ollama.exe
ollama serve
ollama pull gemma3:4b
```

### Q3: 啟動後看不到提示信息

**原因**: 可能使用了舊版本的啟動腳本

**解決**: 確認使用最新版本
```bash
# 檢查腳本內容
type 本地启动.bat | findstr "Gemma"

# 應該看到 "Gemma 3 4B Edition"
```

### Q4: 想使用其他模型

**解決**: 使用命令行參數覆蓋
```bash
# 設置環境變量後啟動
set AGENT_MODEL=llama3.2:3b
python start_server.py
```

## 📈 性能優化提示

### 1. 預先下載模型

在首次使用前，預先下載所有需要的模型：

```bash
# 下載 Gemma 3 4B（用於開發）
ollama pull gemma3:4b

# 下載 Gemma 3 27B（用於生產，可選）
ollama pull gemma3:27b

# 下載 Mistral 7B（備用，可選）
ollama pull mistral:7b
```

### 2. 使用 GPU 加速

確保 NVIDIA 驅動和 CUDA 已正確安裝：

```bash
# 檢查 GPU
nvidia-smi

# 如果看到 GPU 信息，Ollama 會自動使用 GPU
```

### 3. 調整 Ollama 配置

在啟動前設置環境變量：

```batch
REM 調整上下文長度（默認 2048）
set OLLAMA_NUM_CTX=4096

REM 調整並發請求數（默認 1）
set OLLAMA_NUM_PARALLEL=2

ollama serve
```
