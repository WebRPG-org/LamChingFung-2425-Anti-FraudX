## 📋 **使用前檢查清單**

在開始使用前，請確認以下步驟：

### 第 1 步：環境準備

- [ ] 已安裝 Python 3.10+
- [ ] 已安裝 Ollama
- [ ] 已下載模型 `ollama pull gemma3:4b` 或 `gemma3:27b`
- [ ] 已安裝依賴 `pip install -r backend/requirements.txt`

### 第 2 步：配置文件

- [ ] 在 `backend/` 文件夾創建 `.env` 文件
- [ ] 設置 `AGENT_MODEL=gemma3:4b` (或 27b)
- [ ] 設置 `OLLAMA_BASE_URL=http://127.0.0.1:11434`

### 第 3 步：啟動系統

- [ ] Ollama 已運行 (`ollama serve`)
- [ ] 後台已啟動 (`python start_server.py`)
- [ ] 可以訪問 http://localhost:8000
- [ ] API 文檔可用 http://localhost:8000/docs

### 第 4 步：RPG Maker（如果使用）

- [ ] 插件已複製到 `js/plugins/`
- [ ] 插件已在插件管理器中啟用
- [ ] 插件順序正確 (AI_Bridge 在前)
- [ ] 測試連接成功

## ⚠️ **注意事項**

### 1. 端口衝突

- AI-Agent 使用 **port 8000**
- 確保沒有其他程序占用此端口
- Ollama 使用 **port 11434**（默認）

### 2. 模型選擇

- **gemma3:4b**：需要約 4GB VRAM，速度快
- **gemma3:27b**：需要約 16GB VRAM，品質高

### 3. CORS 設置

- 已在 `main.py` 配置為 `allow_origins=["*"]`
- RPG Maker 可以正常訪問
- 生產環境建議限制來源

### 4. 數據存儲

- AI-Agent 訓練：JSON 文件
- RPG Maker 遊戲：SQLite 數據庫
- 兩者獨立，互不衝突
