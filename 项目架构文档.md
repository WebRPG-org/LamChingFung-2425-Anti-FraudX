## 项目概述

### 项目名称
**AI-Agent 多智能体防诈骗模拟系统**

### 核心功能
一个基于多智能体（Multi-Agent）架构的防诈骗教育模拟系统，通过四个AI Agent（骗徒、受骗者、防骗专家、记录分析师）的真实对话模拟，帮助用户理解诈骗手法和防骗策略。

### 技术栈
- **后端框架**: FastAPI (Python 3.10+)
- **AI模型**: Ollama (支持 Gemma3 4B, Mistral 7B)
- **前端**: HTML5 + JavaScript (WebSocket实时通信)
- **数据库**: SQLite (游戏数据), ChromaDB (向量数据库)
- **容器化**: Docker + Docker Compose
- **AI框架**: Google ADK (Agent Development Kit)

### 核心特性
- ✅ **多智能体对话**: 四个独立的AI Agent同时参与对话
- ✅ **实时WebSocket通信**: 对话过程实时显示在前端
- ✅ **动态信任系统**: 受骗者的信任度实时变化，影响对话走向
- ✅ **角色一致性保证**: 严格的角色一致性检查，防止角色混乱
- ✅ **RAG增强**: 专家和骗徒可查询外部知识库
- ✅ **自动训练循环**: 对话结束后自动开始新轮次
- ✅ **军备竞赛系统**: 骗徒与专家通过对抗学习持续进化
- ✅ **性能追踪**: 实时追踪各方表现和效果

## 核心组件

### 1. Agent 系统

#### 1.1 ScammerAgent (骗徒Agent)
**文件**: `backend/agents/scammer.py`

**职责**:
- 扮演专业诈骗者，使用各种诈骗手法
- 通过话术操控受骗者的信任度
- 对抗专家的干预，维持角色一致性

**核心特性**:
- 角色设定: "林志強" - 专业诈骗者
- 支持多种诈骗手法 (WhatsApp诈骗、投资理财、假冒权威等)
- 严格的角色一致性规则 (不能暴露身份、不能提供正确建议)
- 动态策略调整 (根据信任度差距调整话术)

**关键方法**:
- `generate_response()`: 生成诈骗话术

#### 1.2 VictimAgent (受骗者Agent)
**文件**: `backend/agents/victim.py`

**职责**:
- 扮演潜在的受骗者，有三种人格类型
- 维护动态信任系统 (对骗徒和专家的信任度)
- 根据信任度调整反应和行为

**三种人格类型**:
1. **elderly (陈婆婆)**: 72岁退休老人，易信任权威
2. **average (张文轩)**: 35岁文员，谨慎但易被说服
3. **overconfident (李俊杰)**: 28岁IT从业者，自信但易低估风险

**核心特性**:
- 动态信任系统 (0-100分)
- 信任度影响行为决策
- 关键行动词触发 (如"我会转帐"表示被骗)
- 关键拒绝词触发 (如"我唔会信你"表示觉醒)

**信任度规则**:
- `trust_in_scammer >= 100` → 骗徒获胜
- `trust_in_scammer <= 0` 且 `trust_in_expert >= 70` → 专家获胜
- 信任度变化基于对话内容和情绪状态

#### 1.3 ExpertAgent (防骗专家Agent)
**文件**: `backend/agents/expert.py`

**职责**:
- 扮演退休警察"黄sir"，提供防骗建议
- 识别诈骗手法并警告受骗者
- 提供具体、可执行的防骗建议

**核心特性**:
- 角色设定: "黄sir" - 退休警察督察
- 结构化回应模板 (情绪安抚 → 具体判断 → 行动建议)
- RAG增强 (可查询外部知识库)
- 动态策略调整 (根据信任度差距调整话术)

**关键方法**:
- `get_expert_opinion()`: 查询RAG知识库
- `generate_response()`: 生成防骗建议

#### 1.4 RecorderAgent (记录分析师Agent)
**文件**: `backend/agents/recorder.py`

**职责**:
- 记录完整对话过程
- 分析心理动态和关键转折点
- 生成结构化的JSON分析报告

**分析框架 (七层)**:
1. **结果判定**: SUCCESS/FAILURE/PARTIAL
2. **角色分析**: 识别受骗者人格类型
3. **手法分析**: 分析诈骗手法和策略
4. **关键时刻分析**: 识别转折点
5. **失败深度分析**: 多维度分析失败原因 (仅FAILURE时)
6. **改进建议**: 具体、可执行的改进策略
7. **成功因素分析**: 分析成功因素 (仅SUCCESS时)

**输出格式**: 严格JSON格式，包含所有分析字段

### 2. 工具系统

#### 2.1 RoleEnforcer (角色一致性检查器)
**文件**: `backend/utils/role_enforcer.py`

**职责**:
- 检查Agent输出是否违反角色设定
- 检测重复对话
- 检测元语言和脚本输出
- 强制角色一致性重写

**核心功能**:
- `check_scammer_consistency()`: 检查骗徒一致性
- `check_expert_consistency()`: 检查专家一致性
- `check_victim_consistency()`: 检查受骗者一致性
- `detect_repetition_in_history()`: 检测重复对话

#### 2.2 PerformanceTracker (性能追踪器)
**文件**: `backend/utils/performance_tracker.py`

**职责**:
- 实时追踪三方表现 (骗徒、受骗者、专家)
- 追踪信任度变化轨迹
- 生成最终性能报告

**核心数据**:
- `VictimTrustState`: 受骗者信任状态
- `ScammerMetrics`: 骗徒表现指标
- `ExpertMetrics`: 专家表现指标
- `PerformanceReport`: 最终性能报告

#### 2.3 RAGService (RAG服务)
**文件**: `backend/services/rag_service.py`

**职责**:
- 管理向量数据库 (ChromaDB)
- 提供语义检索功能
- 支持专家和骗徒查询外部知识库

**数据源**:
- `data/scraped_alerts.json`: 防诈骗警示文章

**技术栈**:
- ChromaDB: 向量数据库
- SentenceTransformers: 多语言嵌入模型

### 3. 训练与进化系统

#### 3.1 训练数据收集
**目录**: `backend/training_data/`

**数据格式**: JSON文件，包含:
- 完整对话历史
- RecorderAgent分析报告
- 性能追踪报告
- 元数据 (时间戳、回合数、人格类型等)

#### 3.2 军备竞赛系统
**文件**: `backend/scripts/arms_race_system.py`

**职责**:
- 分析训练数据，提取成功/失败模式
- 生成骗徒和专家的进化策略
- 实现对抗学习机制

**流程**:
1. 加载训练数据
2. 分类案例 (成功/失败/部分成功)
3. 提取成功模式 (骗徒成功模式、专家成功模式)
4. 生成进化策略
5. 保存进化数据

#### 3.3 自动化训练
**文件**: `backend/scripts/automated_training.py`

**职责**:
- 将进化知识应用到Agent代码
- 更新Agent指令和策略
- 创建版本备份

**流程**:
1. 加载最新进化数据
2. 更新Agent代码 (scammer.py, expert.py)
3. 创建备份
4. 生成进化报告

## 数据流

### 对话模拟流程

```
1. 用户启动模拟
   │
   ├─→ POST /simulation/start
   │
   ├─→ 生成 simulation_id
   │
   ├─→ 初始化 Agent (Scammer, Victim, Expert, Recorder)
   │
   ├─→ 初始化 PerformanceTracker
   │
   └─→ 建立 WebSocket 连接

2. 对话循环 (turn = 2 to max_turns)
   │
   ├─→ [Turn N] 骗徒发言
   │   │
   │   ├─→ ScammerAgent.generate_response()
   │   │
   │   ├─→ RoleEnforcer.check_scammer_consistency()
   │   │
   │   ├─→ PerformanceTracker.analyze_scammer_turn()
   │   │
   │   └─→ WebSocket 发送消息
   │
   ├─→ [Turn N] 受骗者发言
   │   │
   │   ├─→ 计算信任度变化
   │   │
   │   ├─→ VictimAgent.generate_response()
   │   │
   │   ├─→ RoleEnforcer.check_victim_consistency()
   │   │
   │   ├─→ PerformanceTracker.analyze_victim_response()
   │   │
   │   ├─→ 检查胜利条件:
   │   │   • trust_in_scammer >= 100 → 骗徒获胜
   │   │   • 关键行动词 → 骗徒获胜
   │   │
   │   └─→ WebSocket 发送消息
   │
   ├─→ [Turn N] 专家发言 (可选)
   │   │
   │   ├─→ ExpertAgent.generate_response()
   │   │
   │   ├─→ RAGService.query() (可选)
   │   │
   │   ├─→ RoleEnforcer.check_expert_consistency()
   │   │
   │   ├─→ PerformanceTracker.analyze_expert_turn()
   │   │
   │   ├─→ 检查胜利条件:
   │   │   • trust_in_scammer <= 0 && trust_in_expert >= 70 → 专家获胜
   │   │   • 关键拒绝词 → 专家获胜
   │   │
   │   └─→ WebSocket 发送消息
   │
   └─→ 重复直到:
       • 达到 max_turns
       • 信任度触发胜利条件
       • 用户手动停止

3. 模拟结束
   │
   ├─→ RecorderAgent 生成完整分析
   │
   ├─→ PerformanceTracker 生成最终报告
   │
   ├─→ 保存训练数据 (training_data/*.json)
   │
   ├─→ WebSocket 发送 simulation_end 事件
   │
   └─→ 自动开始新轮次 (如果启用 AUTO_TRAIN_ENABLED)
```

### 信任度变化流程

```
初始状态:
  trust_in_scammer = 50 (根据人格类型调整)
  trust_in_expert = 50

每回合:
  1. 骗徒发言
     ├─→ 分析话术效果
     ├─→ 更新 trust_in_scammer (+10 to +20)
     └─→ 记录到 PerformanceTracker

  2. 受骗者发言
     ├─→ 根据当前信任度调整反应
     ├─→ 检测关键行动词/拒绝词
     └─→ 更新信任度历史

  3. 专家发言
     ├─→ 分析干预效果
     ├─→ 更新 trust_in_scammer (-10 to -20)
     ├─→ 更新 trust_in_expert (+10 to +20)
     └─→ 记录到 PerformanceTracker

胜利条件检查:
  • 骗徒获胜:
    - trust_in_scammer >= 100
    - 检测到关键行动词 ("我会转帐"等)
  
  • 专家获胜:
    - trust_in_scammer <= 0 && trust_in_expert >= 70
    - 检测到关键拒绝词 ("我唔会信你"等)
```

## 关键功能模块

### 1. 模拟引擎 (`simulation_routes.py`)

**核心功能**:
- WebSocket实时通信
- 多Agent对话协调
- 信任度动态计算
- 胜利条件判断
- 自动训练循环

**关键函数**:
- `run_simulation_async()`: 异步模拟主循环
- `websocket_endpoint()`: WebSocket端点处理
- `_generate_recorder_analysis()`: 生成Recorder分析
- `_auto_start_new_round()`: 自动开始新轮次

### 2. 角色一致性系统

**组件**:
- `RoleEnforcer` (`utils/role_enforcer.py`)
- Agent指令中的角色一致性规则

**检查点**:
- 禁止短语检测
- 重复对话检测
- 元语言检测
- 角色混淆检测

**强制执行**:
- 自动重写违反规则的输出
- 提供fallback响应

### 3. 动态信任系统

**组件**:
- `PerformanceTracker` (`utils/performance_tracker.py`)
- `VictimAgent` 信任度计算

**信任度规则**:
- 初始信任度 (根据人格类型)
- 信任度变化规则 (基于对话内容)
- 信任度影响行为
- 胜利条件触发

### 4. RAG增强系统

**组件**:
- `RAGService` (`services/rag_service.py`)
- ChromaDB向量数据库

**工作流程**:
1. 加载防诈骗文章 (`data/scraped_alerts.json`)
2. 文本分块和嵌入
3. 存储到ChromaDB
4. 专家/骗徒查询时进行语义检索

### 5. 自动训练系统

**特性**:
- 对话结束后自动开始新轮次
- 随机选择人格类型和诈骗手法
- WebSocket事件通知前端

**配置**:
- `AUTO_TRAIN_ENABLED`: 环境变量控制开关

## 开发指南

### 环境设置

1. **安装依赖**
   ```bash
   pip install -r backend/requirements.txt
   ```

2. **配置环境变量** (`.env`)
   ```env
   # 基础配置（支持 gemma3:4b, gemma3:27b, mistral:7b）
   AGENT_MODEL=gemma3:4b
   # 或使用 Mistral
   # AGENT_MODEL=mistral:7b
   OLLAMA_BASE_URL=http://127.0.0.1:11434
   
   # 多Agent模型配置 (可选)
   AGENT_MODEL_SCAMMER=gemma3:27b
   AGENT_MODEL_VICTIM=gemma3:27b
   AGENT_MODEL_EXPERT=gemma3:27b
   AGENT_MODEL_RECORDER=gemma3:27b
   # 或使用 Mistral
   # AGENT_MODEL_SCAMMER=mistral:7b
   # AGENT_MODEL_VICTIM=mistral:7b
   # AGENT_MODEL_EXPERT=mistral:7b
   # AGENT_MODEL_RECORDER=mistral:7b
   
   # 自动训练
   AUTO_TRAIN_ENABLED=true
   
   # GPU配置
   FORCE_GPU=1
   ```

3. **启动Ollama**
   ```bash
   ollama serve
   # 拉取 Gemma3 模型
   ollama pull gemma3:4b
   # 或
   ollama pull gemma3:27b
   # 或拉取 Mistral 模型
   ollama pull mistral:7b
   ```

4. **启动后端**
   ```bash
   python start_server.py
   # 或
   docker compose up -d
   ```

## API接口

### REST API

#### 1. 启动模拟
```http
POST /simulation/start
Content-Type: application/json

{
  "victim_persona": "elderly|average|overconfident|random",
  "scam_tactic": "WhatsApp 對話詐騙|random",
  "mode": "fast|demo"
}
```

**响应**:
```json
{
  "simulation_id": "uuid",
  "websocket_url": "ws://localhost:8000/ws/simulation/{simulation_id}"
}
```

#### 2. 停止模拟
```http
POST /simulation/{simulation_id}/stop
```

#### 3. 获取训练状态
```http
GET /api/training/status
```

**响应**:
```json
{
  "is_running": false,
  "total_rounds": 10,
  "current_round": 5
}
```

### WebSocket API

#### 连接
```javascript
const ws = new WebSocket('ws://localhost:8000/ws/simulation/{simulation_id}');
```

#### 事件类型

1. **simulation_start**
   ```json
   {
     "event": "simulation_start",
     "data": {
       "simulation_id": "uuid",
       "victim_persona": "elderly",
       "scam_tactic": "WhatsApp 對話詐騙"
     }
   }
   ```

2. **turn_update**
   ```json
   {
     "event": "turn_update",
     "data": {
       "turn": 2,
       "speaker": "scammer|victim|expert",
       "message": "对话内容",
       "trust_in_scammer": 70,
       "trust_in_expert": 50
     }
   }
   ```

3. **simulation_end**
   ```json
   {
     "event": "simulation_end",
     "data": {
       "outcome": "SUCCESS|FAILURE|PARTIAL",
       "winner": "expert|scammer",
       "total_turns": 15,
       "final_trust": {
         "trust_in_scammer": 0,
         "trust_in_expert": 80
       }
     }
   }
   ```

4. **auto_new_round_starting**
   ```json
   {
     "event": "auto_new_round_starting",
     "data": {
       "new_simulation_id": "uuid",
       "victim_persona": "average",
       "scam_tactic": "投資理財詐騙"
     }
   }
   ```

#### 发送命令

1. **停止对话**
   ```json
   {
     "command": "stop_now"
   }
   ```

### 军备竞赛流程

```
1. 收集训练数据
   │
   ├─→ 运行多轮模拟
   ├─→ 保存训练数据
   └─→ 累积足够数据 (>20个文件)

2. 分析训练数据
   │
   ├─→ 运行 arms_race_system.py
   ├─→ 分类案例 (成功/失败)
   ├─→ 提取成功模式
   └─→ 生成进化策略

3. 应用进化知识
   │
   ├─→ 运行 automated_training.py
   ├─→ 更新 Agent 代码
   ├─→ 创建备份
   └─→ 生成进化报告

4. 测试新版本
   │
   ├─→ 重启服务
   ├─→ 运行新模拟
   └─→ 观察效果变化

5. 重复步骤1-4
```
