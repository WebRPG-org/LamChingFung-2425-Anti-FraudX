## 🔧 **系統配置**

### 1. 環境變數 (.env)

在 `AI-Agent-main/backend/.env` 創建：

```env
# AI 模型設定
AGENT_MODEL=gemma3:4b
# 如果要使用更大的模型：
# AGENT_MODEL=gemma3:27b

# Ollama 連接
OLLAMA_BASE_URL=http://127.0.0.1:11434

# 可選：為每個 Agent 設置不同的端口 (提升VRAM利用率)
# AGENT_MODEL_SCAMMER=gemma3:4b
# AGENT_MODEL_VICTIM=gemma3:4b
# AGENT_MODEL_EXPERT=gemma3:4b
# AGENT_MODEL_RECORDER=gemma3:4b
# OLLAMA_BASE_URL_SCAMMER=http://127.0.0.1:11434
# OLLAMA_BASE_URL_VICTIM=http://127.0.0.1:11435
# OLLAMA_BASE_URL_EXPERT=http://127.0.0.1:11436
# OLLAMA_BASE_URL_RECORDER=http://127.0.0.1:11437
```

### 2. 安裝 Ollama 模型

```bash
# 下載 4B 模型（推薦）
ollama pull gemma3:4b

# 或下載 27B 模型（更強但更慢）
ollama pull gemma3:27b

# 確保 Ollama 運行
ollama serve
```

## 🌐 **API 端點總覽**

## 🎮 **RPG Maker 使用流程**

### 第 1 步：安裝插件

複製文件到 RPG Maker：
```
AI-Agent-main/rpg_maker_plugins/AI_Bridge.js
AI-Agent-main/rpg_maker_plugins/AntiFraudGame.js
```
→ 到 →
```
你的RPG專案/js/plugins/
```

### 第 2 步：啟用插件

1. RPG Maker → 工具 → 插件管理器
2. 添加 `AI_Bridge` 和 `AntiFraudGame`
3. 確保兩個插件都啟用

### 第 3 步：啟動後台

```bash
cd AI-Agent-main
python start_server.py
```

### 第 4 步：在 RPG Maker 中使用

**簡單對話範例**：
```
◆ 插件命令：AI_Bridge > sendToAI
  ├ AI 角色：友善的村民
  ├ 訊息：你好
  ├ 歷史變量：1
  └ 回應變量：2
◆ 等待：30幀
◆ 文字：\V[2]
```

**完整遊戲範例**：
```
◆ 插件命令：AntiFraudGame > startGame
  ├ 角色類型：A
  └ 會話變量：10
◆ 插件命令：AntiFraudGame > startAutoDialogue
  ├ 會話變量：10
  ├ 對話變量：11
  └ 輪數：5
◆ 等待：300幀
◆ 插件命令：AntiFraudGame > getScore
  ├ 會話變量：10
  └ 評分變量：12
```

## 🔄 **循環訓練功能**

### Web UI 操作

1. 訪問 http://localhost:8000
2. 選擇參數（詐騙手法、受騙者類型、模式）
3. 點擊「循環訓練」
4. 系統自動生成訓練數據並存為 JSON

### API 操作

```bash
# 開始循環訓練（加速模式）
curl -X POST http://localhost:8000/api/training/loop/start \
  -H "Content-Type: application/json" \
  -d '{"interval_seconds": 0.1, "mode": "fast"}'

# 查看狀態
curl http://localhost:8000/api/training/status

# 停止訓練
curl -X POST http://localhost:8000/api/training/loop/stop
```

## ⚙️ **性能優化建議**

### 模型選擇

| 模型 | VRAM | 速度 | 品質 | 推薦用途 |
|------|------|------|------|----------|
| gemma3:4b | ~4GB | 快 | 良好 | 開發測試、RPG Maker |
| gemma3:27b | ~16GB | 慢 | 優秀 | 循環訓練、最終產品 |

### 多 Ollama 實例（可選）

為每個 Agent 啟動獨立 Ollama：

```powershell
# PowerShell
$env:OLLAMA_HOST = '127.0.0.1:11434'; ollama serve
$env:OLLAMA_HOST = '127.0.0.1:11435'; ollama serve
$env:OLLAMA_HOST = '127.0.0.1:11436'; ollama serve
$env:OLLAMA_HOST = '127.0.0.1:11437'; ollama serve
```

然後在 `.env` 中配置對應的端口。

### 問題 1：啟動失敗 - 缺少 .env

```
錯誤: .env 文件不存在
```

**解決**：
```bash
cd AI-Agent-main/backend
# 創建 .env 文件（參考上面的配置）
```

### 問題 2：Ollama 連接失敗

```
AI 服務錯誤: Connection refused
```

**解決**：
```bash
# 確保 Ollama 運行
ollama serve

# 測試連接
curl http://localhost:11434/api/tags
```

### 問題 3：RPG Maker 無法連接

```
網路連接失敗
```

**解決**：
1. 確認後台運行在 port 8000
2. 檢查 CORS 設置（已在 main.py 配置）
3. 查看 F8 控制台錯誤訊息

### 問題 4：模型未下載

```
model 'gemma3:4b' not found
```

**解決**：
```bash
ollama pull gemma3:4b
# 或
ollama pull gemma3:27b
```
