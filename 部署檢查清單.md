## 💻 單機部署階段（30台電腦）

### 每台電腦部署前檢查
- [ ] 操作系統：Windows 10/11
- [ ] 硬盤空間：至少 20 GB 可用空間
- [ ] 內存：至少 8 GB RAM（推薦 16 GB）
- [ ] 網絡：**斷開互聯網連接**（確保數據不流出）

### 步驟 1: 安裝基礎軟件
- [ ] 安裝 Ollama
  - 下載：https://ollama.com
  - 安裝到默認位置
  - 驗證安裝：`ollama --version`
- [ ] 安裝 Python 3.10+
  - 下載：https://www.python.org
  - 勾選 "Add Python to PATH"
  - 驗證安裝：`python --version`

### 步驟 2: 複製部署包
- [ ] 從 USB/內網複製部署包到本地
- [ ] 解壓或直接使用部署包目錄
- [ ] 確認目錄結構完整

### 步驟 3: 安裝模型
- [ ] 運行 `scripts\install_models.bat`
- [ ] 確認所有模型導入成功
  ```bash
  ollama list
  ```
- [ ] 驗證模型名稱正確
  - `anti-fraud-expert-20251111_220455`
  - `anti-fraud-scammer-20251111_220455`
  - `gemma3:4b`

### 步驟 4: 配置環境
- [ ] 運行 `scripts\setup_environment.bat`
- [ ] 檢查 `backend\.env` 文件已創建
- [ ] 驗證配置內容：
  ```bash
  type backend\.env
  ```
  確認包含：
  - `OLLAMA_BASE_URL=http://127.0.0.1:11434`
  - `OFFLINE_MODE=true`
  - `AGENT_MODEL_EXPERT=anti-fraud-expert-20251111_220455`
  - `AGENT_MODEL_SCAMMER=anti-fraud-scammer-20251111_220455`

### 步驟 5: 驗證部署
- [ ] 運行 `scripts\verify_deployment.bat`
- [ ] 確認所有檢查通過
- [ ] 記錄電腦名稱和部署時間

### 步驟 6: 測試運行
- [ ] 進入 `code` 目錄
- [ ] 運行 `本地启动.bat`
- [ ] 確認服務啟動成功
  - 訪問 http://localhost:8000
  - 確認前端可以正常顯示
- [ ] 測試一次模擬運行
  - 確認 Agent 可以正常對話
  - 確認模型響應正常

## ⚠️ 常見問題處理

### 問題 1: 模型導入失敗
**症狀**: `ollama load` 失敗
**解決**:
1. 檢查模型文件是否存在
2. 檢查文件完整性（MD5）
3. 重新導出模型

### 問題 2: Ollama 無法啟動
**症狀**: `curl http://localhost:11434` 失敗
**解決**:
1. 檢查端口是否被佔用：`netstat -an | findstr "11434"`
2. 結束舊進程：`taskkill /F /IM ollama.exe`
3. 重新啟動：`ollama serve`

### 問題 3: Python 依賴安裝失敗
**症狀**: `pip install` 失敗
**解決**:
1. 檢查網絡連接（如果需要下載）
2. 使用離線安裝包（如果完全離線）
3. 檢查 Python 版本

### 問題 4: 配置文件未創建
**症狀**: `backend\.env` 不存在
**解決**:
1. 手動複製 `config\.env.template` 到 `backend\.env`
2. 檢查文件權限
3. 重新運行 `setup_environment.bat`

### 問題 5: 服務啟動失敗
**症狀**: `python start_server.py` 報錯
**解決**:
1. 檢查 Python 環境
2. 檢查依賴是否完整
3. 查看錯誤日誌
4. 確認 Ollama 服務運行中
