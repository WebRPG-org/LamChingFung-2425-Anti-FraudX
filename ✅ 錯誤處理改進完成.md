## 💡 **調試建議**

當您看到錯誤時，現在可以：

1. **查看錯誤類型** - 快速了解錯誤性質
   - `TypeError`: 類型錯誤（如參數錯誤）
   - `AttributeError`: 屬性不存在
   - `KeyError`: 字典鍵不存在
   - `ValueError`: 值錯誤
   - `ConnectionError`: 連接錯誤（如 Ollama 無法連接）

2. **查看錯誤訊息** - 了解具體問題

3. **查看 Stack Trace** - 追蹤錯誤來源
   - 第一行：錯誤發生的最外層位置
   - 最後一行：錯誤的實際類型和訊息
   - 中間：調用堆疊

4. **查看 Agent 名稱** - 確定哪個 Agent 出錯
   - 專業騙徒
   - 受騙者
   - 防騙專家
   - 記錄人
